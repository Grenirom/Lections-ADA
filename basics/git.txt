git init - создает git репозиторий из текущей папки (пишется 1 раз в самом начале)
git add . (либо вместо точки любой файл из директории) - добавляет файлы и папки для отслеживания в index (эту команду пишем каждый раз перед командой commit)
git commit -m 'сообщение' - создает новую версию, если были изменения в отслеживающихся файлах


Полный путь от создания локального репозитория, до отправки на удаленный
1) Создаем новый репозиторий на GitHub
2) git init - инициализируем локальный репозиторий, после этой команды в директории должен появиться файл .git, для его просмотра можно выполнить ls -a
3) git add . - добавляем все файлы/папки в индекс git
4) git commit -m 'comment' - коммитим все изменения (сохраняем их)
5) устанавливаем связь через SSH-ключ
    1) генерируем ssh-ключ через команду ssh-keygen
    при создании ключа, в сообщении будет указан путь по которому ключ сохранен. Обычно он выглядит так /home/<имя вашего пользвателя>/.ssh
    2) заходим в папку .ssh, и копируем содержимое файла id_rsa.pub (cat id_rsa.pub)
    3) заходим на GitHub, переходим в setting/настройки
    4) заходим в пункт SSH and GPG keys, и нажимаем кнопку New SSH key
    5) пишем название для ключа (может быть любым) и ниже в поле для ключа, вставляем ключ который скопировали из id_rsa.pub
6) git remote add origin <SSH-ключ с GitHub/либо HTTPS ссылку> - устанавливаем соединение с удаленным репозиторием
7) git push origin main/master - команда которая отправляет все измененния на удаленный репозиторий на GitHub
